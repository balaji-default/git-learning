Git Commands Guide

Git Configuration

git config --global user.name "Your GitHub Username"

git config --global user.email "your-email@example.com"

These commands set up your Git global username and email, used for commit tracking.

Viewing Global Configuration

git config --global user.name - Displays your GitHub username

git config --global user.email - Displays your GitHub email

git config --global --list - Displays both username and email

Local Configuration

git config user.name "Local Username"

git config user.email "local-email@example.com"

If no scope (--global or --local) is mentioned, Git applies changes to the local repository by default.

Viewing Local Configuration

git config --local --list - Lists the local username and email settings

Git Clone Command

git clone <repo-link>

Clones a repository from a remote server (e.g., GitHub) to your local machine.

Viewing Hidden Git Files

To view hidden files, including the .git directory:

  • ls -a (Linux/macOS)
  • Get-ChildItem -Force (Windows PowerShell)

Git Status Command

Displays the current state of your working directory and staging area.

  • Untracked (U): New file, not yet staged.
  • Modified (M): File has been changed but not staged.
  • Staged (A): File is added to the staging area, ready for commit.
  • Committed: File is stored in the repository.

Git Add and Commit

git add filename - Adds a specific file to the staging area.

git add . - Adds all modified and untracked files in the directory.

git commit -m "Meaningful commit message"

Commits the staged changes with a descriptive message.

Good commit messages should be:

  • Concise and meaningful
  • Descriptive of the changes made
  • Consistent with a proper format

Git Push Command

git push origin main

  • Push: Sends committed changes to a remote repository.
  • Origin: The default name for the remote repository.
  • Main: The primary branch where changes are pushed.

git init

Description: Initializes a new Git repository in the local directory.

Usage:

git init

This creates a .git folder, making the directory a Git repository.

git remote add origin <repo-link>

Description: Links your local repository to a remote repository.

Usage:

git remote add origin https://github.com/username/repo.git

Now, origin is the alias for your remote repository.

git remote -v

Description: Displays the remote repositories linked to your local repo.

Usage:

git remote -v

Example output:

origin https://github.com/username/repo.git (fetch)
 origin https://github.com/username/repo.git (push)

git branch

Description: Lists all local branches.

Usage:

git branch

Example output:

* main
    feature-branch

The * indicates the currently active branch.

git branch -M <new-name> vs. git branch -m <new-name>

  • git branch -M main ‚Üí Forces renaming the branch to main, even if the new name already exists.
  • git branch -m main ‚Üí Renames the branch to main only if there is no conflict.

git branch -a

Description: Lists both local and remote branches.

Usage:

git branch -a

Example output:

* main
    feature-branch 
    remotes/origin/main 
    remotes/origin/feature-branch

git branch -M main

Description: Renames the current branch to main.

Usage:

git branch -M main

This is commonly used to rename master to main.

git push origin main

Description: Pushes the main branch to the remote repository.

Usage:

git push origin main

This updates the remote repository with the latest commits from main.

git push -u origin main

Description: Pushes main to the remote and sets it as the upstream branch.

Usage:

git push -u origin main

After this, you can just use:

git push

instead of specifying origin main every time.

Git Branch Management

git branch

Description: Lists all local branches.

git branch

git branch -a

Description: Lists both local and remote branches.

git branch -a

git branch -M main

Description: Renames the current branch to main. (Already explained in the previous section.)

git checkout

Description: Switches to a different branch.

Creating a New Branch

1Ô∏è‚É£ Using git branch (Creates a branch but doesn‚Äôt switch)

Description: This command creates a new branch, but you will still be on the current branch until you switch manually.

Usage:

git branch feature1

This creates a new branch named feature1, but you will remain on the current branch.

2Ô∏è‚É£ Using git checkout -b (Creates and switches to the new branch)

Description: This command creates a new branch and immediately switches to it.

Usage:

git checkout -b feature2

This creates feature2 and switches to it immediately.

3Ô∏è‚É£ Using git switch -c (Modern way to create and switch branches)

Description: This is the preferred method in newer versions of Git for creating and switching branches.

Usage:

git switch -c feature3

This creates feature3 and switches to it.

Checking the current branch

Usage:

git status

Running this command will show which branch you are currently on. If you created the branches using git branch without switching, you will still be on the main branch.
